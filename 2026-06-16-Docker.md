# Docker 심화 명령어 학습 (2026-06-16)

## 이미지 저장/불러오기

### save / load (이미지 단위)
```bash
docker save -o nginx.tar nginx    # 이미지 → tar 파일 저장
docker load -i nginx.tar          # tar 파일 → 이미지 복원
```
- 인터넷 없는 환경에서 이미지 전달할 때 사용
- 레이어 정보 유지됨

### export / import (컨테이너 단위)
```bash
docker export -o webserver.tar webserver  # 컨테이너 → tar 파일
docker import webserver.tar myimg:1.0     # tar 파일 → 이미지
```
- 레이어 하나로 합쳐짐 (히스토리 날아감)
- 실무에서는 save/load 더 많이 씀

### save vs export 차이
| | save | export |
|--|------|--------|
| 대상 | 이미지 | 컨테이너 |
| 레이어 | 유지 | 하나로 합쳐짐 |
| 반대 명령어 | load | import |

## Docker Hub

```bash
docker login                              # Docker Hub 로그인
docker tag nginx parkyoungc/mynginx:1.0  # 업로드할 이름으로 태그
docker push parkyoungc/mynginx:1.0       # Docker Hub에 업로드
docker pull parkyoungc/mynginx:1.0       # Docker Hub에서 다운로드
```

- 이미지 이름이 반드시 `계정명/이미지명:버전` 형식이어야 push 가능
- 실무에서는 Docker Hub 대신 AWS ECR 사용 (개념 동일)

## 볼륨 관리

```bash
docker volume create myvol       # 볼륨 생성
docker volume ls                 # 볼륨 목록
docker volume inspect myvol      # 볼륨 상세 정보 (저장 경로 확인)
docker volume rm myvol           # 볼륨 삭제
```

### 바인드 마운트 vs Docker 볼륨
```bash
# 바인드 마운트 (내가 경로 직접 지정)
docker run -v ~/nginx-html:/usr/share/nginx/html nginx

# Docker 볼륨 (Docker가 경로 관리)
docker run -v myvol:/usr/share/nginx/html nginx
```

| | 바인드 마운트 | Docker 볼륨 |
|--|------------|------------|
| 경로 | 내가 직접 지정 | Docker가 자동 관리 |
| 파일 접근 | 쉬움 | 불편 |
| 용도 | 개발할 때 | 운영(배포)할 때 |

## 네트워크 관리

```bash
docker network create mynet              # 네트워크 생성
docker network ls                        # 네트워크 목록
docker network inspect mynet            # 네트워크 상세 정보
docker run --network mynet --name web nginx  # 네트워크 지정해서 실행
```

### 기본 네트워크 3가지
- `bridge` → 컨테이너 기본 네트워크, docker run 하면 자동 연결
- `host` → 내 PC 네트워크 그대로 사용
- `none` → 네트워크 없음, 완전 격리

### 같은 네트워크면 이름으로 통신 가능
```bash
docker exec -it net1 curl net2  # net1에서 net2로 이름으로 통신
```

## Dockerfile

내가 직접 이미지 만드는 레시피 파일

### 주요 명령어
| 명령어 | 역할 |
|--------|------|
| `FROM` | 베이스 이미지 지정 (필수, 첫줄) |
| `RUN` | 빌드할 때 실행 (패키지 설치 등) |
| `COPY` | 내 파일 → 컨테이너로 복사 |
| `WORKDIR` | 작업 디렉토리 설정 |
| `ENV` | 환경변수 설정 |
| `EXPOSE` | 사용할 포트 명시 |
| `CMD` | 컨테이너 시작할 때 실행할 명령어 |

### 예시
```dockerfile
FROM ubuntu
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y nginx
WORKDIR /var/www/html
COPY index.html .
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 빌드 명령어
```bash
docker build -t mysite:1.0 .          # 이미지 빌드
docker build --no-cache -t mysite .   # 캐시 없이 빌드
```

### 주의
- nginx 공식 이미지 → 웹 루트 `/usr/share/nginx/html/`
- ubuntu에서 nginx 설치 → 웹 루트 `/var/www/html/`
- 베이스 이미지마다 경로 다를 수 있음

## Docker Compose

여러 컨테이너를 파일 하나로 관리

### docker-compose.yml 예시
```yaml
services:
  web:
    image: nginx
    ports:
      - "8081:80"
    volumes:
      - ./html:/usr/share/nginx/html

  db:
    image: mysql:5.7
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: mydb
```

### 주요 명령어
```bash
docker compose up -d          # 전체 실행 (백그라운드)
docker compose down           # 전체 종료 + 삭제
docker compose ps             # 상태 확인
docker compose logs           # 전체 로그
docker compose logs -f        # 실시간 로그
docker compose exec web bash  # 특정 서비스 접속
```

### docker run vs docker compose
```
docker run 여러번   → 명령어 길고 관리 불편
docker compose up  → 파일 하나로 전체 관리, 네트워크 자동 생성
```
