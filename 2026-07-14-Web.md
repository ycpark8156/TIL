- **M05** · posts 배열을 실제 화면에 렌더링, 글쓰기 폼 동작, fetch로 mock 데이터 불러오기

## 핵심 개념

**HTML**
- `<header>/<nav>/<main>/<footer>` — 의미 단위로 영역을 구분하는 시맨틱 태그
- `<form>` + `<label for>` + `<input id>` — id/for를 매칭시켜 라벨과 입력칸 연결

**CSS**
- `class="btn"` → `.btn {}` — 재사용 가능한 스타일 이름표
- `display: flex` — 요소를 가로로 배치하는 레이아웃 모드
- `:hover` — 마우스 오버 시 스타일 변경

**JS**
- `배열 [{...}, {...}]` — 게시글 데이터의 기본 형태. 이후 API가 주고받는 JSON도 같은 구조
- `forEach / map / filter` — 배열을 순회 / 변환 / 필터링하는 대표 메서드
- `document.querySelector('#id')` — HTML 요소를 JS에서 붙잡는 방법
- `addEventListener('submit'/'click', ...)` — 사용자 행동 감지
- `fetch()` + `try/catch` — 데이터 요청 및 실패 대비

## 트러블슈팅

- `fetch('posts.json')` 호출 시 콘솔에 `404 Not Found` 발생
  → 원인: `posts.json` 파일 자체를 안 만들고 넘어감
  → M01에서 배운 "404 = 그 경로에 리소스가 없다"는 개념을 실전에서 그대로 적용해 원인 파악
  → 파일 생성 후 정상 동작 확인

## 느낀 점

- Live Server로 열어야 하는 이유(`file://` vs `http://`)를 M05의 fetch 실습에서 체감함.
  `file://`로 직접 열면 fetch가 막히는 브라우저 보안 정책 때문.
- 문법을 완벽히 암기하기보다 "이 코드가 뭘 하려는지" 흐름을 읽는 데 집중.
  이후 M07(S3)~M11(EC2/RDS)이 실제 목표(클라우드/DevOps)와 직결되므로 그쪽에 힘을 더 쓸 예정.
