## 핵심 개념

**S3 (정적 호스팅)**
- 퍼블릭 액세스 차단은 기본적으로 막혀있음 — 공개 웹사이트로 쓰려면 의도적으로 해제해야 함
- 버킷 정책(JSON)으로 "누가 뭘 할 수 있는지" 규칙 정의 (`s3:GetObject` = 읽기 허용)
- 인덱스 문서를 `index.html`로 지정해야 함 (list.html이 아니라)

**Express + RDS**
- `?` 파라미터 바인딩 필수 — 문자열 직접 연결(`WHERE id=${id}`)은 SQL 인젝션 위험
- `mysql2/promise` + `pool.query()`로 DB 쿼리, `async/await`로 결과 대기
- `.env`로 비밀번호 등 민감정보 분리, `.gitignore`로 GitHub 업로드 차단

**EC2**
- 키페어(.pem)는 최초 1회만 다운로드 가능 — 분실 시 복구 불가
- 보안그룹: SSH(22)는 내 IP만, API 포트(3000)는 전체 공개
- `git clone`으로 GitHub 코드를 서버에 직접 받아옴 → 로컬-서버 간 파일 전달을 Git으로 처리

## 트러블슈팅

1. **RDS DBeaver 접속 실패 "Connect timed out"**
   → 원인: 퍼블릭 액세스는 켜져 있었지만 실제 반영을 위해 "즉시 적용"으로 재수정 필요
   → 해결 후 다음 에러로 진행

2. **"Public Key Retrieval is not allowed"**
   → MySQL8 드라이버 인증 이슈. DBeaver 드라이버 속성에서 `allowPublicKeyRetrieval=true`, `useSSL=false` 추가로 해결

3. **Express에서 RDS 연결 시 "Access denied for user"**
   → `.env`의 DB_PW 값이 실제 RDS 비밀번호와 불일치
   → 비밀번호 재입력으로 해결

4. **DBeaver SQL 스크립트 재실행 시 중복 데이터 발생**
   → `CREATE DATABASE` 중복 에러에서 "건너뛰기(Skip)" 눌렀더니 뒤 INSERT까지 중복 실행됨
   → `DELETE FROM posts WHERE id=?`로 정리. 스크립트 재실행 시 이미 실행된 구문은 주석 처리하는 습관 필요
