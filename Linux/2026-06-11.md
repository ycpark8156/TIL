# 2026-06-11 Linux 웹서버 실습

## 오늘 배운 것

### 웹서버란?
- 브라우저 요청을 받아서 HTML 파일을 돌려주는 프로그램
- Apache, Nginx 두 종류 실습함
- 웹페이지 파일 위치: /var/www/html/

### Apache
- 설치: sudo apt install apache2 -y
- 실행: sudo service apache2 start
- 상태확인: sudo service apache2 status
- 페이지 파일: /var/www/html/index.html

### Nginx
- 설치: sudo apt install nginx -y
- 실행: sudo service nginx start
- 상태확인: sudo service nginx status
- 페이지 파일: /var/www/html/index.nginx-debian.html

### vi 기본 조작
- i : 입력 모드 진입
- ESC : 일반 모드로 복귀
- :wq : 저장 후 종료
- :q! : 저장 없이 강제 종료
- hjkl : 방향키 대신 커서 이동

### grep
- grep "단어" 파일 : 해당 단어 포함된 줄 출력
- grep -n : 줄 번호 포함
- grep -v : 해당 단어 없는 줄만 출력
- grep -i : 대소문자 무시

### pipe
- 명령어 | wc -l : 결과 줄 수 세기
- 앞 명령어 결과를 뒤 명령어 입력으로 넘겨주는 것

## 핵심 개념
- 웹서버 : 브라우저 요청 → HTML 파일 돌려줌
- curl : 터미널에서 웹페이지 내용 확인하는 명령어
- Apache vs Nginx : 둘 다 웹서버, Nginx가 더 가볍고 빠름
- /var/www/html/ : 웹서버가 서비스하는 파일 보관 폴더

## 헷갈렸던 것
- Apache랑 Nginx 동시에 실행하면 포트 충돌남 (둘 다 80번 포트 사용)
- sudo tee vs echo : 관리자 권한 필요한 파일 쓸 때는 sudo tee 사용

## 느낀 점
