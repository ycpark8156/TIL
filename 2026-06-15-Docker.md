# Docker 기초 명령어 학습 (2026-06-15)

## 개념 정리

### 이미지 (Image)
- 컨테이너의 설계도 
- 읽기 전용, 수정 불가
- Docker Hub에서 다운받거나 Dockerfile로 직접 만듦

### 컨테이너 (Container)
- 이미지로 만든 실제 실행 환경
- 삭제하면 안에서 한 작업 날아감 → 볼륨으로 해결

### 볼륨 (Volume)
- 컨테이너 밖(내 PC)에 데이터 저장
- 컨테이너 삭제해도 데이터 유지됨
- `-v 내PC폴더:컨테이너폴더` 로 연결

## 이미지 명령어

```bash
docker search nginx          # Docker Hub에서 이미지 검색
docker pull nginx            # 이미지 다운로드
docker images                # 로컬 이미지 목록
docker inspect nginx         # 이미지 상세 정보 (포트, 환경변수 등)
docker history nginx         # 이미지 레이어 히스토리 (아래→위 순서)
docker tag nginx myimg:1.0   # 이미지에 이름/버전 태그 추가 (복사 아님, 이름표만 추가)
docker rmi myimg:1.0         # 이미지 삭제
```

---

## 컨테이너 명령어

```bash
# 실행
docker run -d --name webserver -p 8080:80 nginx
# -d : 백그라운드 실행
# --name : 이름 지정
# -p 8080:80 : 내PC포트:컨테이너포트 연결

# 목록
docker ps        # 실행중인 컨테이너만
docker ps -a     # 멈춘 것까지 전부

# 접속
docker exec -it webserver /bin/bash
# exit 쳐도 컨테이너 유지됨

# attach
docker attach webserver
# exit 치면 컨테이너 꺼질 수 있음

# 로그
docker logs webserver      # 로그 확인
docker logs -f webserver   # 실시간 로그

# 정지/삭제
docker stop webserver
docker rm webserver
docker rm -f webserver     # 실행중인 컨테이너 강제 삭제

# 일괄 처리
docker stop $(docker ps -a -q)   # 전체 정지
docker rm $(docker ps -a -q)     # 전체 삭제
```

## 이미지 저장/배포

```bash
docker commit webserver mywebserver:1.0  # 컨테이너 → 이미지로 저장

docker login                             # Docker Hub 로그인
docker tag myimg:1.0 username/myimg:1.0  # 태그 추가
docker push username/myimg:1.0           # Docker Hub 업로드
docker pull username/myimg:1.0           # Docker Hub 다운로드
```

## 볼륨 실습

```bash
mkdir ~/nginx-html
echo "Hello YOUNGCHAN" > ~/nginx-html/index.html

docker run -d --name webserver -p 8080:80 \
  -v ~/nginx-html:/usr/share/nginx/html nginx
# ~/nginx-html 폴더 ↔ 컨테이너 /usr/share/nginx/html 연결
# 컨테이너 삭제해도 ~/nginx-html 데이터 유지됨
```

## 핵심 포인트

| 개념 | 설명 |
|------|------|
| `>` | 파일 덮어쓰기 (리다이렉션) |
| `>>` | 파일 이어쓰기 |
| `-p 왼:오` | 왼쪽=내PC포트, 오른쪽=컨테이너포트 |
| `exec` vs `attach` | exec은 exit해도 컨테이너 유지, attach는 꺼질 수 있음 |
| `tag` | 복사 아님, 같은 이미지에 이름표만 추가 |
| `commit` | 컨테이너 현재 상태를 이미지로 저장 |
