# 2026-06-12 Linux 실습

## 권한 관리

### chmod - 파일 권한 변경
- chmod 777 파일 : 모든 사람 읽기+쓰기+실행
- chmod 644 파일 : 소유자만 쓰기, 나머지 읽기만
- chmod 600 파일 : 소유자만 읽기 (pem 키 파일)
- chmod +x 파일 : 실행 권한 추가

### 권한 숫자 의미
- 4 : 읽기 (r)
- 2 : 쓰기 (w)
- 1 : 실행 (x)
- 7 = 4+2+1 : 전부 허용
- 6 = 4+2 : 읽기+쓰기
- 4 = 4 : 읽기만

### chown - 소유자 변경
- sudo chown root 파일 : 소유자를 root로 변경
- sudo chown user1231 파일 : 소유자를 user1231로 변경

## 프로세스 관리
- ps aux : 실행 중인 프로세스 전체 목록
- ps aux | grep nginx : nginx 프로세스만 필터링
- kill PID : 프로세스 종료
- kill -9 PID : 프로세스 강제 종료
- top : 실시간 프로세스 모니터링
- sleep 1000 & : 백그라운드로 프로세스 실행

## 네트워크 명령어
- ss -tlnp : 열려있는 포트 확인
- sudo netstat -tlnp : 포트 + 프로세스 상세 확인
- sudo netstat -tlnp | grep :80 : 80번 포트 누가 쓰는지 확인
- curl -I http://주소 : 서버 응답 헤더 확인
- wget URL : 파일 다운로드

### HTTP 상태코드
- 200 : 정상
- 304 : 캐시 사용 (파일 안 바뀜)
- 404 : 파일 없음
- 500 : 서버 에러

## 실시간 로그 분석
- tail 파일 : 마지막 10줄 출력
- tail -f 파일 : 실시간으로 로그 모니터링
- tail -f 파일 | grep "404" : 실시간으로 404만 필터링

### Nginx 로그 파일 위치
- /var/log/nginx/access.log : 접속 로그
- /var/log/nginx/error.log : 에러 로그

## 쉘 스크립트
- 명령어들을 파일에 모아서 한 번에 실행
- 첫 줄에 #!/bin/bash 써야 함
- chmod +x 파일 로 실행 권한 준 후 실행
- ./파일.sh 또는 ~/경로/파일.sh 로 실행

## 핵심 개념
- 포트 충돌 : Apache랑 Nginx 동시에 80번 포트 못 씀
- pem 키 파일은 chmod 600 으로 설정해야 EC2 접속 가능
- tail -f + grep 조합으로 실시간 에러 모니터링 가능
- 쉘 스크립트로 반복 작업 자동화 가능
